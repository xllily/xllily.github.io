Concepts

Introduction

webpack is a module bundler, and it can creates a “graph” of all of your application’s dependencies.
It is necessary to first understand Four Core Concepts(entry,output,loader and plugins) before get started!

  • entry tells webpack where to start and what to bundle.
  • output tells webpack where to put bundled various resources and how to treat bundled code.
  • loaders transform files ( .js(ECMAScript 2015), .jsx, .ts, .coffee .sass, etc. ) into .js(ECMAScript 2011) or transfrom files ( .css, .html, .jpg, etc. ) into modules, and then add all of them into “dependency graph”.
  • plugins can extend a variety of functionality, customize the webpack build process.

Entry Points

  • Single Entry Syntax

    entry: string | Array<sting>

The single entry syntax is a shorthand for object syntax :

Passing an array of file paths will inject multiple dependent files together graph their dependencies into one “chunk”.

  • Object Syntax

    entry: { entryChunkName : string | Array<string> }

The object syntax is verbose,but it is the most scalable way.
“Scalable webpack configurations” that means can be reused and combined with other partial configurations. ( More info : specialized tools like webpack-merge ).

Output

Output options tell webpack how to write the compiled files to disk.
There’re two properties be required, filename and path .
Only one output configuration can be specified.

output.path
The output directory(required).

output.filename
Specifies the name of each output file on disk.
single entry

multiple entries
If configuration creates more than a single “chunk”, then should ensure each file has a unique name.
[name] name of chunk
[hash] hash of compilation
[chunkhash] hash of chunk

More info:

  • __dirname refers to the directory where the webpack.config.js lives.
  • more other options of Output.

Loaders

They are functions (running in Node.js) that take the source as the parameter and return the new source after transformation.
rules be used instead of loaders since webpack v2.1.0-beta.23

Features

  • Loaders can be chained.
  • Loaders can be synchronous or asynchronous.
  • Loaders can do everything that’s possible in Node.js
  • Loaders accept query parameters.( This can be used to pass configuration to the loader )
  • Loaders can also be configured with an options object.
  • Loaders can produce additional arbitrary files.

Plugins

Plugins are the backbone of webpack.
Plugins can give loaders more features,can do anything else that a loader cannot do.
A webpack plugin is a javascript object that has apply method.The apply method will be called by the webpack compiler,so the plugin can access to the entire compilation lifecycle.

Configuration

Webpack’s configuration file( webpack.config.js ) exports an object. Webpack will process the object based the properties defined by the object.

webpack.config.js is a standard Node.js CommonJS module, so more flexible than using .json to configure. Technically you can do almost all you want to do, but not recommended following things:

  • Access CLI arguments, when using the webpack CLI (instead write your own CLI, or use --env)
  • Export non-deterministic values (calling webpack twice should result in the same output files)
  • Write long configurations (instead split the configuration into multiple files)
